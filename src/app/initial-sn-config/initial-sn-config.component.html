1. Initial Global Update Set

Load and apply the initial global update set.
  1. Navigate to System Update Sets ⇒ Retrieved Update Sets
    Navigation and related links example
  2. Click on Import Update Set from XML under Related Links
  3. Click on the Choose File button and upload the sys_remote_update_set_Initial_Setup.xml File in the downloads folder of this documentation.
    Example of file upload dialog
  4. Once it is loaded, open the update set and click the Preview Update Set button.
    Example of uploaded upset set
    ⇒
    Example of preview success popup
  5. After it is validated, click the Commit Update Set to apply it.
    Example of commit success popup

2. Import Utility Application

  1. Navigate to System Applications ⇒ Studio. Once you click on Studio, the application studio will open in separate browser tab.
    Example of navigation
  2. Click on the Import from Source Control button in the popup dialog.
    Example of Select Application dialog
  3. Enter credentials for connecting to the associated Git repository.
    Example of git credentials dialog
  4. Progress will be indicated as it imports the application. Once it is finished, click the Select Application button.
    Example of application import progress
    ⇒
    Example of application import success
  5. You should now see the imported application listed. Select the application in order to put your session in the correct scope, and then close out the Application Studio browser tab.
    Example of selected application
    ⇒
    Example of application selection dialog
  6. In the previous browser tab, your session will now be in the Utility application scope. You will want to be in this scope for the next section. Navigate to System Properties ⇒ General Properties and change the SDLC Stage setting to reflect the stage (production, uat, dev, etc) of the current ServiceNow instance.
  7. Navigate to ??? ⇒ Fix Scripts and select the Fix Script named Initial Army Config.
  8. This script will take a few minutes to execute, so it will be your choice if you wish to run it in the foreground or you can run it in the background and check the status, later. If you run it in the background, you can check its status by navigating to ??? ⇒ Progress Workers.
  9. Once this script is finished, the colors scheme will be configured for the current environment. If you run it in the background, you can check its status by navigating to ??? ⇒ Progress Workers.

3. Initial Config

  1. Navigate to System Properties ⇒ General Properties. If you are in the "Util" application scope, you will see the following message: This is intentional. If you do not see this message, ensure you are in "Util" application scope.
  2. Navigate to System Definition ⇒ Fix Scripts.
  3. Open the fix script named Initial Army Config.
  4. Click Run Fix Script.
  5. Click Proceed if you wish to wait for the script to finish; otherwise, click Proceed in Background to let it run in the background. This script can take several minutes to finish.

4. Bulk Plugin Activation

  1. Navigate to System Definition ⇒ Fix Scripts.
  2. Open the fix script named Army Bulk Plugin Activation.
  3. Click Run Fix Script.
  4. Click Proceed if you wish to wait for the script to finish; otherwise, click Proceed in Background to let it run in the background. This script can take a long time to finish.
