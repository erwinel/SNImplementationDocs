1. Initial Global Update Set

Load and apply the initial global update set to prepare the instance.
  1. Navigate to System Update Sets ⇒ Retrieved Update Sets
  2. Click on Import Update Set from XML under Related Links
  3. Click on the Choose File button and upload the sys_remote_update_set_Initial_Setup.xml File in the downloads folder of this documentation.
  4. Once it is loaded, open the update set and click the Preview Update Set button. ⇒
  5. After it is validated, click the Commit Update Set to apply it.

2. Import Utility Application

In this step you will be importing a utility application which includes many utility framework functionality to support other scripts. This will also create a custom system property which can be used by automated processes to determine whether the current instance is for production, UAT, Testing, Development or if it is a sandbox instance.
  1. Navigate to System Applications ⇒ Studio. Once you click on Studio, the application studio will open in separate browser tab.
  2. Click on the Import from Source Control button in the popup dialog.
  3. Enter credentials for connecting to the associated Git repository.
  4. Progress will be indicated as it imports the application. Once it is finished, click the Select Application button. ⇒
  5. You should now see the imported application listed. Select the application in order to put your session in the correct scope, and then close out the Application Studio browser tab. ⇒

3. Initial Config

In this step you will specify whether the current instance is for production, UAT, Testing, Development or if it is a sandbox instance, and then you will execute a fix script to apply the corresponding color theme and other SDLC-related settings, such as the logo heading text.
  1. Navigate to System Properties ⇒ Basic Configuration. If you are in the "Util" application scope, you will see the following message: This is intentional. It is important to ensure you are in the "Util" application scope, otherwise the next change will not be saved.
  2. Set SDLC Config Stage to the ServiceNow SDLC stage for the current server.
  3. Change to the "Global" application scope.
  4. Navigate to System Definition ⇒ Fix Scripts.
  5. Open the fix script named Initial Army Config.
  6. Click Run Fix Script.
  7. Click Proceed if you wish to wait for the script to finish; otherwise, click Proceed in Background to let it run in the background. This script can take several minutes to finish. If you chose to let it run in the foreground, the logged output that occurred during script execution will be displayed when it is finished. At the bottom of the output, you should see a line that indicates it has succeeded.
    Refresh the browser page to see the applied color scheme.

4. Upload logo image

To automate configuration settings for graphical images, it would require significant overhead. Therefore, the logo image will need to be uploaded manually.
  1. Ensure you are in the "Global" application scope.
  2. Navigate to System Properties ⇒ Basic Configuration.
  3. Scroll to the "Banner Image" property and click on the + icon to upload the banner image.
  4. Upload the logo image corresponding to the SDLC stage you previously configured. You can download an image from one of the following links:
    • Production
    • UAT
    • Test
    • Dev
    • Sandbox
    • Out-of-the-box

5. Bulk Plugin Activation

In this step you will launch a fix script which will activate plugins in the background. Plugin activation progress can be monitored from the progress workers list.
  1. Navigate to System Definition ⇒ Fix Scripts.
  2. Open the fix script named Army Bulk Plugin Activation.
  3. Click Run Fix Script.
  4. Click Proceed if you wish to wait for the script to finish; otherwise, click Proceed in Background to let it run in the background. This script can take a long time to finish.
To monitor the progress of plugin activations, navigate to System Diagnostics ⇒ Progress Workers.

6. Import Group Names Registry Application

The Group Names Registry applications provides a way of synchronizing User Group references between ServiceNow instances, such as Dev, Test and Production.
