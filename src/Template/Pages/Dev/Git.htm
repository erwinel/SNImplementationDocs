Credential Management

Set credential helper

The following will configure git to use the windows credential helper: 
git config --global credential.helper wincred

Set login for credential helper

When using the windows credential helper, and you need to use alternate login credentials, you may find that it continues to try to authenticate using the same credentials, regardless of the erwine.email setting. To force the windows credential helper to use alternate login credentials, you can apply the following global configuration setting: 
git config --global credential.microsoft.visualstudio.com.username username
Applying this setting to the local repository seemed to have no effect the last time it was tried.

Viewing Credentials

From the command line run the command control /name Microsoft.CredentialManager

Set default email address

Using your personal private email address that is associated with the github account: 
git config --global user.email email@address
Using github-provided no-reply email address that is associated with the github account: 
git config --global user.email userid+loginName@users.noreply.github.com

Set default user name

This sets the default user name for all repositories where the user name is not configured (this is not the github username. Use user.email to associate with a specific account): 
git config --global user.name user name

Managing Repositories

Push existing repository

First, create a new, totally empty remote github repository. 
git remote add origin url_to_remote_repo
git push -u origin master

Clone/check out remote repository

cd parentDirectory
git clone --recurse-submodules url directoryName

Managing Branches

Deleting a branch

If you wish to delete a branch, you can use the following command: 
git branch -D name_of_branch

Common Local Git Repository Synchronization Issues

No upstream branch

Many times, when a new local clone of a remote repository is created, sync and push commands will fail, saying there is no upstream branch. This can be fixed with the following command, replacing "remoteName" with the name of the remote repository and "remoteBranch" with the name of the remote branch (which is typically the name of the current branch): 
git branch --set-upstream-to remoteName/remoteBranch
So, if your remote repository name is "origin", and your current branch is "master", then the actual command would be: 
git branch --set-upstream-to origin/master

Getting the name of the remote repository

If you aren't sure what the name of the remote repository is, simply use the following command: 
git remote

Getting branch names

Typically, you would be able to see the name of the current branch in the lower-left corner of the Visual Studio Code editor window. Alternatively, you can see what the name of the currently checked out branch is by using the following command: 
git branch
You don't have to have a branch checked out in order to configure the upstream branch. To see a list of all the branch names use: 
git branch -a
